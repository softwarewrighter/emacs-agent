# Demo Scripts Overview

## Core Demos (Keep These)

### `demo-minibuffer.el`
Tests if LLM infers `(interactive)` declarations from phrases like "M-x" and "mini-buffer".
- **Success Rate:** 71% interactive functions created
- **Run:** `emacs --batch -l emacs-agent-mcp.el -l demo-minibuffer.el`

### `demo-autonomous.el`  
Tests LLM's autonomous decision-making with minimal guidance.
- **Key Finding:** LLM chooses single-file organization and sensible naming
- **Run:** `emacs --batch -l emacs-agent-mcp.el -l demo-autonomous.el`

### `demo-working.el`
Basic working demo with comprehensive logging.
- **Purpose:** Proves the concept with direct requests
- **Run:** `emacs --batch -l emacs-agent-mcp.el -l demo-working.el`

### `run-math-demo.sh`
Simple shell wrapper for math library creation.
- **Purpose:** Easy first demo for new users
- **Run:** `./run-math-demo.sh`

## Supporting Files

### `emacs-agent-mcp.el`
The main MCP-compliant implementation. This is the heart of the system.

### `test-general-agent.el`
Test suite using ERT framework.
- **Run:** `./validate-all.sh`

### `demo-mcp-strategies.el`
Tests different prompting strategies (reference implementation).

## Test Output

The `test_output/` directory contains logs and generated files from demo runs:
- `comprehensive.log` - Detailed execution logs
- `*.el` files - Generated by Gemini
- `decisions.log` - Shows LLM's autonomous decisions

Keep some test outputs for review, especially those showing successful tool use.
